# 🗺️ Дорожная карта развития Shop-bot

> Последнее обновление: 2024-08-21

## 🔴 Критические задачи (0-1 неделя)
1. **Cryptography**: заменить использование `md5` на `sha256` или `uuid4` в `utils/image_storage.py`.
2. **Безопасность исполнения**: убрать все `assert` из прод-кода, заменить на явные проверки и исключения.
3. **Транзакции БД**: обернуть операции пополнения/списания кошелька и оформление заказа в `BEGIN IMMEDIATE…COMMIT`.
4. **Docker-данные**: добавить volume `data:/app/data` в `docker-compose.yml`, чтобы сохранять базу данных между перезапусками.

## 🟠 Задачи следующего спринта (1-2 недели)
1. **Сервисный слой**: вынести бизнес-логику из `handlers` в `services`.
2. **Миграции**: внедрить Alembic, описать существующие таблицы как revision `0001`.
3. **Типизация**: включить строгий `mypy` для пакета `utils`, настроить gradual для остальных.
4. **Тесты**:
   - покрытие CRUD операций товаров и корзины;
   - мок Telegram API для handlers.
5. **CI оптимизация**: кэширование pip, matrix-шаги lint + security.

## 🟡 Среднесрочные улучшения (3-4 недели)
1. **Docker multi-stage**: builder + runtime, избавление от `gcc` в финальном образе.
2. **Healthcheck**: добавить `/health` endpoint и `HEALTHCHECK` в Dockerfile.
3. **Индексы БД**: `CREATE INDEX IF NOT EXISTS idx_cart_cid ON cart(cid);` и аналогичные.
4. **README**: расширить раздел API / FAQ.
5. **Pre-commit**: flake8-bugbear, flake8-annotations, black.

## 🟢 Долгосрочные цели (1-3 месяца)
1. **Микросервисы**: вынести платёжную логику в отдельный сервис (FastAPI).
2. **Кэш Redis**: хранить сессии и анти-флуд счётчики.
3. **Мониторинг**: Prometheus + Grafana, алерты на ошибки.
4. **CI/CD**: автоматический docker-release в GHCR и деплой на Railway.
5. **Bot UX**: AB-тестирование сценариев, мультиязычность.
